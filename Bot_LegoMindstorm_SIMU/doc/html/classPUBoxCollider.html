◆ updatePUAffector()

void PUBoxCollider::updatePUAffector ( PUParticle3D *  particle,
float  deltaTime 
)
overridevirtual

Collision detection is a two-step. First, we determine whether the particle is now colliding. If it is, the particle is re-positioned. However, a timeElapsed value is used, which is not the same as the one that was used at the moment before the particle was colliding. Therefore, we rather want to predict particle collision in front. This probably isn't the fastest solution. The same approach was used for the other colliders.

Reimplemented from PUAffector.
